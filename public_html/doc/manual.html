osweb.runner
This is the main object class used for running an Osexp experiment. It is pre-defined in the osweb library and is called using the notation osweb.runner. It has several properties and methods which are accessible within Javascript. For an example of this usage see the ‘index .html’ file in the ‘public_html’ folder of the project.

Properties

runner.data Array []
Property which contains the result data after an experiment is executed. By default this is an empty array. Content of this property depends on the data logging settings of the Osexp file that has been executed.

runner.debug boolean false
Property which enables or disables the internal debugger. If enabled all debug messages are logged to the internal Javascript log system within the browser. By default the debugger is disabled. This property can be set by the context parameter of the run method.

runner.experiment Object null
When running an experiment this property contains the experiment object tree which is build from the Osexp file.

runner.status Number 0
Property that contains the current status of the runner. This status changes during the execution of an experiment or due to user interruption or a fatal error. The following values are defined.

0 No experiment is loaded in the runner.
1 Experiment is loaded successfully and ready to run.
2 Experiment is running.
3 Experiment is finished.
4 Runner is stopped due to an error.
5 Runner is stopped due to a forced exit.

Methods

osweb.runner.exit()
This methods can be used to gracefully exit an running experiment. When this method is called a break flag is set to true and the experiment will stop directly.

osweb.runner.run(content, context)
This method runs an OpenSesame experiment file within an HTML page. The content parameter must contain an HTML canvas object or a valid id name of an HTML canvas object on which the experiment output is shown. The context parameter is a JSON based object which contains information about the experiment to run. The following context parameters are currently defined.

context.debug boolean false
Parameter which enables or disables the internal debugger. If enabled all debug messages are logged to the internal Javascript log system within the browser. By default the debugger is disabled.

context.introclick booleean true
Parameter which enables or disables the click-on-screen procedure to start on experiment. Mainly used for setting the focus on the experiment canvas, the user is requested to click on the screen to start the experiment. This can also be used to force a user defined pause before starting the real experiment.

context.introscreen boolean true
Parameter which enabled or disabled the introduction screen. The introduction screen is used to show feedback about the loading process of an experiment. If this parameters is set to false only a blank screen is show. By default the introduction screen is shown.

context.onfinished function callback null
Can contain a callback function which is executed after the experiment is finished. This function contains two parameters ‘data’ and ‘session_data’ which contain the result data of the experiment and session information about the client computer.

context.source string/fileobject/url null
Must contains a valid OpenSesame soure. This can be a string bases script file, a Javascript file object or a valid url to an OpenSesame file on a server location.

context.target url null
Must contain a valid target location to which the result data is send. This can be a URL link with parameters. Data is send using a post request. If this parameter is empty no data is stored.
